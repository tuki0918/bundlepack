# Cross-platform fixtures

These packages are generated by the native macOS implementation and opened by
`Windows/BundlePack.Core.Tests`. They live outside either platform's test
directory because they are part of the shared interoperability contract.

- `macos-unencrypted.bundlepack` is a standard ZIP-compatible package.
- `macos-encrypted.bundlepack` uses the encrypted container documented in
  [Docs/FORMAT.md](../../../Docs/FORMAT.md).
- `macos-unicode-password.bundlepack` verifies that canonically equivalent composed and decomposed passwords interoperate.
- fixture password: `BundlePack-Compatibility-2026!`
- Unicode fixture password: `Café-Compatibility-2026!` (stored in decomposed form by the generator and opened in composed form by Windows tests)

The password and payload are intentionally public test data. Never copy production or private data into this directory.

Regenerate the macOS fixtures after an intentional format change:

```sh
./macOS/Scripts/generate-compatibility-fixtures.sh
```

Windows CI creates the matching `windows-*.bundlepack` files, including a
Unicode-password fixture created in composed form and opened in decomposed form
by macOS. Compatibility and password normalization are therefore tested in both
directions.
